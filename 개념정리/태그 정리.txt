인라인은 제약이 많은편 여백이나 높이 변하지 않음. 블록은 변경 가능.
인라인 요소 안에 블록 요소를 넣을 수 없다.

<블록(상자) 요소 => 아래로 쌓는다.>
1. div : 특별한 의미가 없는 구분을 위한 요소 (Division)
2. h1 : 제목을 의미하는 요소 (Heading) => 숫자가 작을수록 중요도 큼.
3. p : 문장을 의미하는 요소 (paragraph)
4. ul : 순서가 필요없는 목록의 집합 (Unordered List)
5. li :  목록 내 각 항목 (List item)

<인라인(글자) 요소 => 옆으로 채운다. >
1. img : 이미지를 삽입하는 요소 (Image)
2. a : 다른/같은 페이지로 이동하는 하이퍼 링크를 지정하는 요소 (Anchor)
   => target = "_blank" 하이퍼 링크로 브라우저를 열 때, 새 탭으로 열 것이다.
3. span : 특별한 의미가 없는 구분을 위한 요소 (이 부분만 색 변경 하고 싶다)
4. br : 줄바꿈 요소 (Break) <br/> 사용해주기
5. label : 라벨 가능 요소(input)의 제목

<테이블 요소>
1. table : 표 요소, 행row과 열colum의 집합
 => tr 태그로 먼저 행을 나타냄
 => td 태그로 열을 나타냄. (Table Data)

<Inline-block 요소> 인라인요소이긴 한데, 상자요소의 특성을 갖는다.
=> 글자요소 처럼 수평으로 쌓이지만, 가로/세로, 여백 조절이 가능하다.
1. input : 사용자가 데이터를 입력하는 요소
   placeholder = "이름을 입력하세요!" (사용자가 입력할 값(데이터)의 힌트)
   disabled = 입력요소 비활성화 (입력이 불가능)
   type=checkbox => 사용자에게 체크 여부를 입력 받음. checked로 미리체크 가능
